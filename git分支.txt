git 分支创建和切换使用
  关于git项目管理分支部分
项目中有时需要我们创建多个分支来进行项目管理和开发，所以掌握git的基本使用是很有必要的 
我们首先可以在本地创建一个本地分支
1.创建一个文件夹  里面随便增加一个文件 （作为git commit 不为空的条件）
2.类似于上传主分支  确切说是我们想要上传代码  第一步就需要git init  初始化项目 
3.git add . 提交修改过得所有文件 
4.因为是分支操作  但是也有主分支和其他分支操作  所以我们必须要先将项目上传到本地仓库的主分支下  
才能进行接下来的其他分支操作   此时当git commit -m '随意写，例如创建分支wh'
 如果报错 Not a valid object name: 'master'. 原因就是我们没有 git commit 
5.提交到本地仓库后  我们就可以创建新的分支了  git branch wh （创建一个新的分支wh）我们可以Git branch 查看当前所有的分支  也可以不查看
有时可能git branch 什么都不显示  这个无所谓  
6.接下来我们需要把分支切换到我们需要操作的分支上面  所以  git checkout wh  切换到wh分支上  
7.接着git  remote add origin ‘git 项目地址’
8.git push origin wh 记住  此地不是master  因为不是主分支操作  所以不是master

有的时候分支太多  就需要我们进行分支代码管理啦 但是有的分支可能是我们不需要的  此时我们可以进行删除分支
首先git branch 
删除本地分支
git branch -D 分支名  -D ===--delete/-d
删除本地的远程分支
git branch -r -D origin/分支名
删除远程git服务器上的分支
git push origin -d 分支名
